# QiReader TUI: Subscriptions

本文固定订阅页的布局、入口和与文章页的衔接约定。

## 范围

- 订阅分组与订阅源列表
- 分组展开/折叠
- 未读数显示
- 从订阅页进入文章列表页

## 页面角色

订阅页是插件主入口，也是文章域之前的发现页。

- 页面继续使用 KOReader 原生 `Menu`
- 左侧 `state` 列负责分组展开/折叠
- 右侧 `mandatory` 列显示未读数
- 点击 `全部`、分组标题或订阅源行，进入对应文章列表页

## 顶部菜单

顶部左侧按钮打开锚定二级菜单，仅保留：

- `Account`
- `Unread only: On/Off`

点击 `Account` 后，直接打开统一账号弹窗：

- 标题固定为 `QiReader account`
- 弹窗内包含 `Email`、`Password` 两个输入框
- 底部固定三个按钮：`Cancel` / `Log out` / `Log in`
- 已登录时，`Log out` 可用，`Log in` 灰显
- 未登录时，`Log in` 可用，`Log out` 灰显
- 不再额外进入账号详情页
- 不再额外弹登出确认框

## 列表交互

- `全部` 没有折叠按钮，点击整行直接进入全部文章列表
- 业务分组左侧 `state` 区域负责折叠/展开
- 业务分组标题点击进入该分组的文章列表
- 订阅源行点击进入该订阅源的文章列表
- 从文章列表返回后，应恢复订阅页原先的展开状态和可见页位置
- 未登录且当前没有可显示订阅时，订阅页显示为空列表，不额外显示 `Not logged in` 占位行

## 列表样式

- 分组项粗体
- 订阅源常规字重
- 未读数显示为纯数字，不带括号
- 已读状态可灰显，但不改变排序
- 折叠/展开图标与后续标题之间保留较宽间隙，避免视觉过挤

## 状态约定

- 订阅页自身状态保持 `loading`、`groups`、`error`、`closed`
- 登录 / 登出流程都在同一个账号弹窗内完成，不单独占据整页状态
- 登录态通过账号弹窗按钮的可用/灰显状态反映，不额外插入状态说明行
- 返回订阅页时，不重新创建整个页面，而是恢复原先菜单状态
