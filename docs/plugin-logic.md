# QiReader Plugin Handoff

本文用于交接当前 `qireader.koplugin` 的实际状态。

只保留已经实现、已经确认、或者会直接影响后续排查路径的信息。
旧的方案讨论、TUI 草图和过时判断不再放在这里。

## 当前范围

插件入口仍在 KOReader FileManager 主菜单。

当前代码已经覆盖两层界面：

- 订阅页：基于 KOReader 原生 `Menu`
- 文章页：基于自定义全屏 widget

订阅页已具备：

- 登录/登出
- 拉取订阅源与分组
- 未读数展示
- 仅显示未读过滤
- 分组展开/折叠
- 进入单个订阅或分组的文章列表

文章页代码已接入：

- 按订阅源或分组拉取文章流
- 分页翻页
- 未读过滤
- 旧到新 / 新到旧排序
- 翻页标记已读
- `!readlater` 标签驱动的“稍后”状态切换
- 打开文章正文

## 当前接口假设

当前实现不是基于正式插件协议，而是基于 Web API 探测结果。

已落到代码的接口面：

- `/subscriptions`
- `/markers/unread/counts`
- `/streams/{streamId}`
- `/entry-contents`
- `/markers/reads`
- `/tags`
- `/entries/.../tags/...`

登录态依赖 Cookie。

本地只保存：

- `cookie`
- `user`
- `subscriptions_version`
- 列表相关设置项

本地不保存密码。

`401` 的处理约定已经固定：

- 清空本地登录态
- 清空当前分组数据
- 关闭文章页
- 回到订阅页

不要再恢复“独立登录首页”的旧模型。

## 已确认的实现约定

订阅页继续使用 KOReader 原生 `Menu`，不重写整页列表框架。

文章页标题直接显示当前订阅源标题或分组标题，不显示额外的 `Articles`、排序提示等辅助文案。

文章列表第二行文案为：

- `日期 | 订阅源标题`

右侧动作区文案固定为：

- `稍后`

“稍后阅读”不是独立布尔字段，而是特殊标签 `!readlater`。

已读/未读状态当前约定为：

- 不再显示左侧圆点
- 已读项通过灰显表达

文章页设置菜单当前包含：

- 只显示未读
- 从旧到新排序
- 翻页标记为已读
- 每页列表项
- 列表项字号

`每页列表项` 的含义已经固定：

- 设备一屏内目标显示多少条

`列表项字号` 的含义已经固定：

- 调整文章标题字号

当空间允许时，标题可以多行显示。
但不是无限增长。
每个列表项的实际高度由设备可用高度和“每页列表项”共同决定。

## 当前真实问题

文章页在设备实测时仍存在布局错误。

已观察到的现象：

- 左半边大面积空白
- 标题和副标题整体偏到右侧
- 每页显示条目数和设置值不稳定
- 右侧动作区在截图中未按预期出现

相关截图：

- 工作区根目录 `IMG_0191.HEIC`

这不是纯接口问题，而是 KOReader 运行时布局问题。

## 当前最大不确定点

当前最需要优先确认的不是 API，而是运行时是否真的加载了最新文章页代码。

需要特别注意：

- KOReader Lua 模块存在 `require` 缓存
- 仅覆盖设备文件，不等于当前运行中的 KOReader 会重新加载模块

如果设备没有完整重启 KOReader，可能仍在使用旧版 `qireader.articlelist`。

因此下一轮排查必须先确认：

1. 设备当前运行的 KOReader 是否重启过
2. 当前文章页是否确实走到最新 `qireader/articlelist.lua`
3. 若已重启仍复现，再继续查布局树

## 下一步建议

后续工作建议按这个顺序推进：

1. 先确认设备侧 KOReader 完整重启后问题是否仍复现
2. 必要时在 `qireader/articlelist.lua` 加最小调试标记，证明运行时加载的是新代码
3. 若确认已加载新代码，再继续对照 KOReader 原生 `Menu` / `vocabbuilder` 查布局树
4. 优先解决文章页布局，再讨论进一步交互优化

不要继续扩展接口面、缓存设计或离线模型，除非文章页先稳定显示。
